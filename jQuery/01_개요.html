jQuery 개요

* 개요

기존에 자바스크립트 언어만으로 구현하기에
복잡했던 구문 작성들을 간소화하기 위해서 "존 레식"에 의해 개발된
JavaScript 기반 라이브러리 (유용한 기능들의 모음집)
즉 JavaScript 언어로 유용한 함수나 내용들이 이미 만들어져있는 (정의되어있는) 모음
복잡한 자바스크립트 크도를 짧고 쉽게 개발할 수 있다
-jQuery 장점
1) DOM요소와 관련된 스크립트를 보다 쉽게 구현할 수 있다
2) AJAX (비동기 통신), 이벤트 처리 등 폭넓게 지원한다
3) jQuery 와 관련된 확장형 플러그인, 오픈 API를 지원한다

jQuery 라이브러리 연결 방법

head태그 내부에 script 태그로 jQuery관련 .js파일 가져오기
오프라인/온라인 방법이 있다.
-jQuery 공식 사이트 : 공식 사이트

1.jQuery 라이브러리 직접 다운로드 후 경로 지정 사용 (오프라인 방식)
-.js 파일을 내가 직접 다운로드 받아 저장한 후 해당 저장된 경로를 작성하는 방법


unCompressed : 가독성을 유지하는 형태 (개발자가 확인할 수 있도록) 일반 JS 파일

Compressed : 배포전용 - 주속, 들여쓰기, 띄어쓰기 등을 최소화하여 용량을 줄인 형태 2.CDN(Content Delievery Network) 를 통한 연결(온라인 방식)
-라이브러리를 제공하는 사이트 url을 제시해서 파일 경로로 지정


jQuery 구문 작성하기

jQuery 구문으로 DOM 요소를 다루는 구문을 기본적으로 작성할것이기 때문에
jQuery 명령은 문서상에 요소들이 다 로드 (만들어진 후) 실행되어야한다.



-순수 자바스크립트에서의 해당 내용은 
        window.onload = function(){
            실행내용;
        }
        onload 함수 : 페이지가 다 로딩되면 자동으로 실행되는 함수
        단, JavaScript에서의 window.onload구문은 해당 문서의 요소들도
        다 로드되고 또한 연동되어있는 외부 문서들도 다 가지고 온 뒤에 실행된다.
        window.onload는 해당 문서에서 단 한번만 작성 가능하다.
-jQuery에서는 해당 내용은 3가지 구문으로 제공한다. 
        1) jQuery(document).ready(function(){
            실행내용
        })

        2) $(document).ready(function(){
            실행내용;
        })

        3) $(function(){

        })

        jQuery에서의 해당 function은 DOM요소들만 다 로드되면 바로 실행된다.
        여러번 기술 가능하다
