form 태그 내의 submit 버튼 클릭 시 해당 form 내에서 작성됨
사용자가 입력한 값들을 서버에 전달하며 요청을 보내는 역할

form action="" method=""
-action 속성: 해당 form 내에 사용자가 입력한 값들을 전달하며 요청할 서버의 경로 제시 -method 속성: 해당 전송방식을 지정하는 속성(get(기본값),post)
-get 방식: 요청시 사용자가 입력한 값들이 url에 노출되는 방식으로 조회의 용도로 많이 사용한다 -post 방식: 요청시 사용자가 입력한 값들이 url에 노출되지 않는 방식으로 정보변경, 추가등의 용도로 사용한다. form 태그 내의 제출(submit) 버튼 클릭시 action에 지정한 서버로 method에 작성한
요청 방식으로 사용자가 입력한 값들을 전달한다(key&value 형태로)

라디오 버튼(type="radio")

제시한 여러개의 값들 중에서 오로지 한개만 선택해야 할때

성별:

체크박스(type="checkbox")

-제시한 여러개의 값들 중에서 여러개를 선택 가능한 경우
-현재 선택된(checked) 값들이 넘어간다
-넘기고자 하는 value값을 명시해야한다

select태그와 option태그

-사용자에게 직접 입력받은 것이 아니라 내가 제시한 여러 옵션들 중에
하나를 선택할 수 있게 하는 태그
-제출시 현재 선택된 (selelcted) 값이 넘어간다.
각각의 option에 value값을 명시했을 경우 value값이 전송된다.
만일, 명시하지 않았을 경우에는 기본적으로 option태그의 content영역의 값이 넘어간다(시작 종료태그 사이 택스트)
